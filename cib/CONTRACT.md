# Контракт блока cib

Сюда вписывай ручки, которые твой блок отдаёт наружу. Соседи по команде
видят только этот файл — не код. Если ручка изменилась или появилась новая —
обнови этот файл, иначе сосед о ней не узнает.

## Что я отдаю наружу

### GET /health
Проверка живости. Возвращает `{status, team, block, commit, backend_url, products}`.

### GET /products
Каталог продуктов команды. Возвращает `{total, items: [продукты]}`. Каждый
продукт — объект как минимум с `{id, kind, name}`; депозитные/кредитные могут
иметь `rate_pct`. В каталоге сейчас три продукта: дебетовая карта, срочный
депозит и кредитная карта Classic (`credit-card-classic`, ставка 21.9%,
лимит 30 000 – 500 000 ₽, грейс 55 дней).

### POST /credit/decide
Решение по заявке на кредитную карту. Принимает JSON
`{client_id, amount_rub}`. Внутри спрашивает у backend
`GET /clients/{client_id}` и считает посильный платёж по простому правилу
(нет просрочек, положительный баланс, доход покрывает три месячных
платежа при сроке 24 месяца). Возвращает
`{approved, client_id, requested_rub, approved_rub, rate_pct, product_id,
reason, explanation}`. `rate_pct` и `product_id` заполнены только при
одобрении. `explanation` — короткий текст для клиента, сгенерирован LLM
(или нейтральный fallback, если LLM не сконфигурирован).

Коды ошибок: `404` если backend не нашёл клиента, `502` если backend
недоступен.

### GET /
HTML с каталогом продуктов и карточкой кредитной карты. Для человека, не
для других блоков.

## Кого я зову у соседей

- backend: `GET /clients/{client_id}` — нужен для решения по кредиту
- retail: я никого не зову у retail — это retail зовёт меня

## Где работает блок локально

`http://localhost:8002` (порт фиксируется docker-compose).
